# ADR 007: 평가 방법론 + ChatGPT vs Gemini 비교 결과

- **결정일**: 2026-05-25
- **상태**: 컨펌 완료 (2026-05-25)
- **관련 Task**: Task 11

---

## 최종 결정 (방법론)

| 항목 | 값 |
|---|---|
| 평가셋 | Golden Dataset v1 — 11문항 (답있음 8 + 환각 3), `eval/golden_dataset.json` |
| 비교 설계 | **통제 변수 1개** — 임베더 고정(openai-small) + temp 고정(0) → **LLM만** gpt-4o-mini vs gemini-2.5-flash |
| 자동 지표 | latency / tokens / cost / recall@k / 거부율 / citation coverage |
| 수동 지표 | 정확도(1-5) / Noise(0-4) / Citation 일치(Y/N) |
| 러너 | `eval/evaluate.py` |

> **왜 통제 설계?** "같은 검색결과를 주고 누가 더 잘 답하나"를 보려면 변수가 LLM 하나여야 함. 임베더까지 다르면 차이의 원인(LLM vs 임베더)을 분리 못 함.

---

## 5요소

- **배경**: MVP 완성 후 "OpenAI vs Gemini 어느 게 나은가"를 정량 측정. 막연한 "잘 되네"가 아니라 데이터 기반 결론.
- **임팩트**: 모델 선택 근거 확보 + Golden Dataset v1 seed 구축(v2에서 50→200 확장 → 회귀 측정 기반).
- **대안 비교**: 6조합 전체 vs 2조합. → 수동 채점 부담(60건) + 변수 혼재 때문에 **2조합 통제 비교** 선택.
- **동작 원리**: 11문항 × 2조합 = 22회. `prompt | llm`로 AIMessage 받아 토큰/비용 확보. recall@k는 hit의 source URL idxno로 측정.
- **사이드 이펙트**: temp=0이라 거의 결정적(재현 가능). 수동 채점은 주관 개입 → 1인 채점 한계(v2: 복수 채점자/Ragas).

## 결론

### 종합: Gemini가 평균 정확도 높음 (4.5 vs 3.94), 단 상황별로 갈림

| 질문 유형 | 우세 | 이유 |
|---|---|---|
| 사실 / 종합 (q1-q7) | **Gemini** | 맥락·완성도↑, 청크 순위에 얽매이지 않고 relevance 우선 배치, 인용 충실(coverage 1.0) |
| **희박/부분정보 (q8 삼성)** | **ChatGPT** | 🔄 **역전.** Gemini가 "삼성 AI안경"을 너무 좁게 해석해 주변 맥락 누락. ChatGPT는 인접 정보 살림 |
| 환각 (q9-q11) | 동일 | 둘 다 완벽 거부 (거부 정확도 1.0) |

### 용도별 추천
- **빠른 훑기 / 비용 민감 / 간결 선호** → **ChatGPT** (3.4s, $0.0033, 114토큰)
- **깊은 이해 / 맥락 풍부 / 인용 충실** → **Gemini** (단 ~3.4배 비용, ~2배 길이, 느림)

### ⭐ 제품 목적 기준 재해석 (중요)
이 프로젝트의 원래 목적은 *"읽을 시간이 없어서 만든 뉴스 요약 RAG"* — **빠른 소화**다.
그 기준으로 보면 **Gemini의 "완성도 우위"가 오히려 목적을 배신할 수 있다**:
- Gemini는 2배 길고 3.4배 비쌈 → "빠르게 훑는다"는 목적과 충돌. 장문이 답을 묻을 수 있음.
- 즉 **"평균 정확도 Gemini 우세(4.5)"는 이 제품엔 오해의 소지.** 제품 목적(skimming)을 1순위로 두면 **ChatGPT의 간결함이 더 적합**할 수 있다.
- **결론: "어느 모델이 객관적으로 낫다"가 아니라, "제품 목적에 맞느냐"가 기준.** → 사용자가 **빠른 요약 / 깊은 이해 모드를 직접 선택**하게 하는 것이 최선 (v2).

### ⚠️ Caveat (공정성 + 채점 방법론 한계)
- **체급 불일치**: gemini-2.5-flash는 gpt-4o-mini보다 **신세대**. Gemini가 품질로 이긴 게 'provider' 덕인지 '모델 세대' 덕인지 단정 불가. 동세대 비교는 v2.
- **정확도 vs 완성도 혼재** ⚠️: 채점에서 Gemini에 일관되게 5, ChatGPT에 4-4.5를 줬는데, 이는 **"정확도(틀린 게 없나)"보다 "완성도(많이 다뤘나)"를 잰 것에 가깝다.** 둘 다 사실 오류가 없었다면 정확도는 대등하고, 차이는 완성도/스타일. → 점수 격차는 "정확도 격차"가 아니라 "완성도·스타일 선호"로 해석해야 함.
- **Citation 일치 = article-level만 확인** ⚠️: 채점 시 "내용이 그 기사에서 나왔으면 청크 번호 무관하게 맞다"고 봤음(article-level). 하지만 citation laundering은 **claim-level**(그 문장의 주장이 실제 인용 내용과 일치하나)에서 발생 → 모든 Y가 laundering을 배제하지 못함. 특히 Gemini는 문장마다 인용(coverage 1.0)이라 claim-level 미검증분이 더 많음. **coverage 1.0 ≠ 정확도 1.0.** claim-level 검증은 v2.
- **1인 수동 채점**: 주관 개입. 정확도 점수는 참고치.
- **샘플 작음**: 11문항. Golden Dataset 확장(v2) 후 재측정 권장.

### 핵심 인사이트
1. **Recall 동일** = 통제 설계 성공. LLM 차이가 순수하게 측정됨.
2. **"무조건 Gemini 우세"가 아님** — q8 역전이 보여주듯 **희박 정보엔 Gemini의 엄격함이 독**. 상황별 선택이 정답.
3. **노이즈는 검색 탓**(두 모델 공통) — q2/q3/q8의 노이즈는 Dense 임베딩의 키워드 블리드(같은 회사 다른 주제) 한계. LLM이 아니라 retrieval 문제 → v2 Hybrid/MMR.
4. **같은 기사 다른 청크 번호 혼란** (q3 사용자 피드백) — `[1][2][3]`이 같은 기사면 인용이 헷갈림 → MMR(중복 제거) v2 근거.
5. **종합형 질문의 recall 부족** — q4(구글 종합)는 기대 기사 3개 중 2개만 검색(recall 0.667). k=4가 다기사 종합엔 부족 → 동적 k(종합형은 크게) v2 근거.
6. **제품 목적이 "승자"를 바꾼다** — 평균 정확도는 Gemini지만, 원래 목적인 "빠른 소화"엔 Gemini의 장황함이 역효과. "객관적 우열"보다 "목적 적합성"이 기준이어야 함 (위 재해석 참고).

## 사고 과정 (결정의 여정)

- **Gemini 모델 교체**: 처음 `gemini-2.0-flash` 지정 → 실행 시 "신규 사용자 미제공" 404 (ADR 006에서 예견). `gemini-2.5-flash`로 교체. 가격도 2.0(무료 가정)에서 2.5 유료로 갱신.
- **체급 vs 가격 매치 고민**: gemini-2.5-flash가 gpt-4o-mini보다 비싸고 신세대 → "large(gpt-4o)로 체급 맞출까" 검토했으나, "각사 기본 저가 모델 비교"가 실용적이고 비용도 지표라 **mini vs flash 유지 + caveat 명시** 결정.
- **q10 부분 답변 딜레마**: "구글 vs 오픈AI 누가 이길까"는 예측(거부 정답)이지만 구글 팩트는 데이터에 있음. "팩트는 답하고 예측만 거절"이 더 친절하나, 의견 질문에 모델이 슬쩍 답할 위험↑. MVP는 통째 거부 유지(precision-first), **부분 답변은 v2**. (temperature는 거부 제어 레버 아님 — fluency용)

## v2-backlog 연계
- **⭐ 답변 모드 선택 (빠른 요약 / 깊은 이해)** — 사용자가 용도를 직접 고르게. 빠른 모드=간결 프롬프트(또는 ChatGPT), 깊은 모드=상세 프롬프트(또는 Gemini). "어느 모델이 나으냐"를 사용자 선택으로 해소.
- **부분 답변 (partial answering)** — 답 가능한 사실은 답하고 추측/없는 부분만 명시적 거절 (q10)
- **Citation claim-level 검증** — article-level이 아니라 문장 주장↔인용 내용 일치 자동 검증 (laundering 탐지)
- **동세대 LLM 비교** — 체급 맞춰 재평가 (공정성)
- **Golden Dataset 확장** — 11 → 50 → 200, 복수 채점자 / Ragas 자동화
- **동적 k** — q4 recall 0.667처럼 종합형 질문은 k 부족 → 질문 유형별 k 가변
- **MMR / Hybrid** — 노이즈(키워드 블리드) + 같은 기사 중복 청크 해소
